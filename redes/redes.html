Redes de Computadores: A Tecia que Conecta o Mundo

As redes de computadores são o sistema nervoso da era digital, permitindo a comunicação e compartilhamento de informações em escala global. Compreender seus fundamentos é essencial para qualquer profissional de tecnologia.

1. Conceitos Fundamentais

O que são Redes de Computadores?

São conjuntos de dispositivos interconectados que compartilham recursos e informações através de protocolos de comunicação estabelecidos.

O que é uma Rede mínima?

Uma rede mínima consiste em pelo menos dois dispositivos conectados, podendo ser dois computadores diretamente ligados por um cabo ou via wireless.

O que é a Internet? O que a compõe?

A Internet é uma rede global de redes interconectadas, composta por:

  • Provedores de serviços (ISPs)
  • Backbones de fibra óptica
  • Data centers e servidores
  • Dispositivos de usuários finais
  • Roteadores e switches
  • Protocolos de comunicação (TCP/IP)

O que é a Web?

A World Wide Web (WWW) é um sistema de informação que opera sobre a Internet, utilizando o protocolo HTTP para acesso a documentos interligados por hyperlinks.

2. Infraestrutura Global: Mapa de Fibra Óptica

O que é o Mapa de Fibra Óptica?

É a representação visual da rede global de cabos submarinos e terrestres que formam o backbone da Internet, conectando continentes e países.

Características da Infraestrutura Global:

  • Mais de 1,3 milhões de km de cabos submarinos
  • Conexões transoceânicas entre todos os continentes
  • Velocidades que chegam a terabits por segundo
  • Infraestrutura crítica para economia global
  • Investimentos bilionários em expansão contínua

4. Protocolos e Modelos de Comunicação

O que é um Protocolo de Comunicação?

É um conjunto de regras e padrões que governam a comunicação entre dispositivos em uma rede, definindo formato, timing, sequenciamento e controle de erros.

Modelo OSI

Modelo de referência de 7 camadas:

  1. Física: Meio físico de transmissão
  2. Enlace: Controle de acesso ao meio
  3. Rede: Roteamento e endereçamento
  4. Transporte: Controle de fluxo e erro
  5. Sessão: Controle de diálogo
  6. Apresentação: Formatação de dados
  7. Aplicação: Interface com usuário

Modelo TCP/IP

Modelo prático de 4 camadas:

  1. Acesso à Rede: Combina física e enlace
  2. Internet: Roteamento (IP)
  3. Transporte: Comunicação fim-a-fim (TCP/UDP)
  4. Aplicação: Serviços aos usuários

5. Protocolos IP: IPv4 e IPv6

Protocolo IPv4

Características:

  • Endereços de 32 bits (4.3 bilhões de endereços)
  • Notação decimal pontuada (ex: 192.168.1.1)
  • Esgotamento de endereços
  • Uso de NAT para extensão
  • Protocolo dominante por décadas

Protocolo IPv6

Características:

  • Endereços de 128 bits (3.4×10³⁸ endereços)
  • Notação hexadecimal (ex: 2001:0db8:85a3::8a2e:0370:7334)
  • Segurança integrada (IPsec)
  • Melhor eficiência de roteamento
  • Transição gradual do IPv4

6. Configuração de IP e TCP em Node.js

Configuração Básica de Servidor

// Servidor HTTP básico
const http = require('http');

const server = http.createServer((req, res) => {
    res.writeHead(200, {'Content-Type': 'text/plain'});
    res.end('Servidor rodando!\n');
});

// Configurando IP e porta
const PORT = 3000;
const HOST = '0.0.0.0'; // Escuta em todas as interfaces

server.listen(PORT, HOST, () => {
    console.log(`Servidor rodando em http://${HOST}:${PORT}`);
});

Configurações Avançadas

  • HOST: '0.0.0.0' para acesso externo, 'localhost' para local
  • PORT: Portas comuns: 80 (HTTP), 443 (HTTPS), 3000, 8080
  • TCP Keep-Alive: Configurações de timeout e reutilização
  • Socket Options: Configurações de buffer e backlog

7. Protocolos HTTP e HTTPS

Protocolo HTTP

HyperText Transfer Protocol

  • Protocolo de aplicação para a Web
  • Comunicação em texto plano
  • Porta padrão: 80
  • Métodos: GET, POST, PUT, DELETE, etc.
  • Stateless (sem estado)
  • Vulnerável a interceptação

Protocolo HTTPS

HTTP Secure

  • HTTP sobre camada de segurança (SSL/TLS)
  • Comunicação criptografada
  • Porta padrão: 443
  • Autenticação do servidor
  • Integridade dos dados
  • Obrigatório para sites modernos

Características:

  • Roteamento robusto
  • Middleware extensível
  • Comunidade ativa
  • Fácil integração com outros módulos

Exemplo Básico:

const app = express();

app.get('/', (req, res) => {
    res.send('Hello World com Express!');

app.listen(3000, () => {
    console.log('Servidor Express rodando na porta 3000');
});
