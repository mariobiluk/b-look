Banco de Dados: O Coração dos Sistemas de Informação

Os bancos de dados são a espinha dorsal de qualquer sistema moderno. Eles permitem armazenar, organizar, recuperar e gerenciar dados de forma eficiente, garantindo integridade e segurança das informações.

1. O que é um SGBD? Tipos e Aplicações

Sistema Gerenciador de Banco de Dados (SGBD)

Um SGBD é um software que permite criar, manter e manipular bancos de dados. Ele atua como uma interface entre os usuários/aplicações e o banco de dados físico.

Tipos de SGBD

Relacional (SQL)

Organiza dados em tabelas relacionadas. Ex: MySQL, PostgreSQL, Oracle

Não-Relacional (NoSQL)

Armazena dados em formatos flexíveis. Ex: MongoDB, Cassandra, Redis

Orientado a Objetos

Armazena objetos complexos. Ex: db4o, Versant

Em Memória

Armazena dados na memória RAM. Ex: Redis, Memcached

Aplicações

  • Sistemas bancários e financeiros
  • E-commerce e comércio eletrônico
  • Redes sociais
  • Sistemas de saúde
  • Gestão empresarial (ERP)

3. Dicionário de Dados e Tipos de Dados

O que é um Dicionário de Dados?

É um repositório centralizado de metadados que descreve a estrutura, relações e características dos dados armazenados no banco de dados.

Informações Contidas

  • Nomes de tabelas e colunas
  • Tipos de dados de cada campo
  • Restrições e chaves
  • Relações entre tabelas
  • Descrições dos campos

Tipos de Dados em Bancos de Dados

Numéricos

  • INTEGER - Números inteiros
  • DECIMAL - Números decimais
  • FLOAT - Números de ponto flutuante

Texto

  • VARCHAR - Texto de tamanho variável
  • CHAR - Texto de tamanho fixo
  • TEXT - Texto longo

Data/Hora

  • DATE - Data
  • TIME - Hora
  • DATETIME - Data e hora

Especiais

  • BOOLEAN - Valores verdadeiro/falso
  • BLOB - Dados binários
  • JSON - Dados estruturados

4. Modelagem de Banco de Dados: DER e MER

O que é Modelagem de Dados?

É o processo de criar um modelo abstrato que representa a estrutura, relações e restrições dos dados de um sistema.

Modelo DER (Diagrama Entidade-Relacionamento)

Representa graficamente as entidades (objetos do mundo real), seus atributos e os relacionamentos entre elas.

Modelo MER (Modelo Entidade-Relacionamento)

É a representação conceitual do DER, focando na estrutura lógica dos dados sem considerar implementação física.

Elementos do DER/MER

Entidade

Objeto do mundo real (ex: Cliente, Produto)

Atributo

Característica da entidade (ex: Nome, Preço)

Relacionamento

Associação entre entidades (ex: Compra, Pertence)

Cardinalidade

Quantidade de relacionamentos (1:1, 1:N, N:M)

5. Exemplo de Diagrama DER

Sistema de Biblioteca - DER

Exemplo de diagrama para um sistema de gerenciamento de biblioteca:

Entidades Principais

  • Livro: ISBN, Título, Autor, Ano
  • Autor: ID, Nome, Nacionalidade
  • Usuário: ID, Nome, Email, Telefone
  • Empréstimo: ID, Data_Empréstimo, Data_Devolução

Relacionamentos

  • Autor escreve Livro (1:N)
  • Usuário realiza Empréstimo (1:N)
  • Livro está em Empréstimo (1:1)
